Step by Step

Git Workflow

Follow the essential Git commands from repository initialization to pushing code on GitHub

Initialize Repository

git init

Creates a new Git repository in your project directory. This is always the first step.

Working Directory

Edit files

Make changes to your files. Git tracks all modifications in your working directory.

Staging Area

git add .

Stage your changes for commit. Select which modifications to include in your next commit.

Commit Changes

git commit -m "message"

Save your staged changes with a descriptive message. Creates a permanent snapshot.

Create Branch

git branch feature

Create isolated branches for features. Keep your main branch clean and stable.

Merge Branches

git merge feature

Combine branches together. Integrate completed features into main branch.

Push to GitHub

git push origin main

Upload your commits to GitHub. Share your code with the team and backup remotely.

Branching Model

Branch Strategy

Organize your development workflow using purpose-driven branches

Feature Branch

feature/login-page

Develop new features in isolation. Created from the main branch and merged after completion.

Test Branch

test/integration

Run integration and QA tests before merging to main. Ensures code quality and stability.

Bugfix Branch

bugfix/auth-error

Fix bugs and issues quickly. Prioritized for immediate merge to maintain stability.

Experiment Branch

experiment/new-ui

Try new ideas without risk. Safe space for prototyping that may or may not be merged.

Conflict Resolution

Merge Conflicts

Understand and resolve merge conflicts when they occur

What is a Merge Conflict?

Occurs when Git cannot automatically merge changes because the same line is modified differently in multiple branches.

Why Does It Happen?

Two developers edit the same lines, or one deletes a file while another modifies it. Git needs human decision.
